Privacy Policy

1 · Overview & Scope

This Privacy Policy describes how Christian Omania, a California sole proprietor operating as Abreo ("I," "me," or "the application"), collects, uses, and protects financial data accessed through this private, single-user personal finance and property management tool.

The application is operated exclusively by its owner for the purpose of managing personal real estate finances.

The application is not directed to children under 13 and does not knowingly collect information from them.

2 · Architecture & Data Residency

The application is local-first. All financial data is stored exclusively in a database file on the owner's personal hardware, which is protected by full-disk encryption. No financial data is uploaded to, transmitted to, or stored on any remote server, cloud database, or third-party storage provider under the application's control.

The application is not accessible via the public internet.

3 · Data Collected via Plaid

The application uses Plaid Inc. ("Plaid") to connect to the owner's own financial institutions. By initiating a connection in Plaid Link, the owner authorizes Plaid to access the owner's financial data at those institutions and to transmit it to the application for the purposes described below.

Data types accessed: Transaction history and categorization · Account balances · Investment holdings and transactions (securities, positions, cost basis) · Mortgage liabilities (balance, interest rate, escrow, payment schedule) · Credit card liabilities (APR, statement balance, minimum payment, due date)

Financial credentials (usernames and passwords) are never seen or stored by the application. All credential handling occurs exclusively within the Plaid Link module. Use of Plaid is also subject to the Plaid End User Privacy Policy.

4 · Purpose of Data Use

Data accessed through Plaid is used solely for the following personal purposes:

· Categorizing rental income and property-related expenses to assist with Schedule E tax preparation
· Monitoring mortgage balances, amortization, and escrow accounts
· Monitoring investment holdings, transactions, and cost basis to maintain an accurate net-worth picture
· Monitoring credit card balances, interest rates, and payment due dates for budgeting
· Tracking cash flow across operating and personal accounts

Data retrieved via Plaid is not anonymized, aggregated with other users' data, or used for analytics of any kind. It is never used for marketing, profiling, credit decisions, or any commercial purpose.

5 · Data Security

At rest: All data, including Plaid API access tokens, is stored on a machine protected by full-disk encryption (AES-256). No financial data exists outside of this encrypted local environment.

In transit: All communications with the Plaid API are conducted exclusively over TLS 1.2 or higher.

Access control: The application runs on a password-protected machine accessible only to the owner. No other individuals have access to the application, the database, or any data it contains.

Regulatory scope: Financial data retrieved via Plaid constitutes "nonpublic personal information" under the Gramm-Leach-Bliley Act (GLBA). Such information is handled in accordance with GLBA's safeguarding requirements and is not disclosed to any non-affiliated third party for purposes other than those described in this policy.

6 · No Third-Party Sharing or Sale of Data

The application does not sell, rent, license, share, or otherwise disclose any personal or financial data to any third party for any purpose. The sole exception is the transmission of data to and from Plaid as described in Section 3, which is necessary for the application to function.

7 · Data Retention & Deletion

The owner may delete all stored financial data at any time by removing the local database file. Disconnecting a bank account through the Plaid Link interface will revoke the application's access token and cease all future data retrieval for that institution.

No automated cloud backups of the application's database are configured under the application's control.

8 · California Privacy Rights (CCPA/CPRA)

Because the sole user of this application is its operator, and all data is stored exclusively on the operator's personal device, the application does not process personal information of any individual other than the operator. CCPA/CPRA rights therefore apply only to the operator, who exercises them directly through control of the local data store.

9 · Changes to This Policy

This policy will be updated if the application's scope or data handling materially changes. Because the operator is the sole user of this application, no external notification is required; changes take effect on the "Last updated" date shown in this document.
